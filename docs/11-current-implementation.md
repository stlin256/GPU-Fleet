# 当前实现说明

本文记录当前仓库中已经落地的实现，避免文档规划和代码状态脱节。

## 已实现

- Go module：`gpufleet`。
- 服务端命令：`cmd/gpufleet-server`。
- 客户端命令：`cmd/gpufleet-agent`。
- Agent 使用 `nvidia-smi` 只读采集 NVIDIA GPU 指标，优先采集扩展字段并在驱动不支持时回退到基础字段集。
- Agent 支持 `--print` 本地采集调试模式。
- Agent 当前字段包含显存空闲/保留、显存利用率、温度上限、功耗上限、图形/显存/SM/视频时钟、PCIe 当前/最大链路、VBIOS、Compute、显示状态、驱动模型、ECC/MIG 和时钟限速原因等；不支持字段为空。
- Agent 使用 HMAC-SHA256 签名主动上报。
- 服务端校验设备 ID、时间戳、nonce 和签名。
- 服务端拒绝重放 nonce。
- 服务端支持 gzip 请求体，并限制解压后的请求体大小。
- 服务端使用 gzip JSONL 分段文件保存时序指标。
- 服务端写入前按保留期清理旧分段，再检查磁盘保护阈值。
- 服务端默认保留 800MiB 磁盘空闲空间，低于阈值返回 `507`。
- 服务端使用 JSON 文件保存 Web 密码凭据、设备、服务配置和审计元数据。
- 服务端支持设备创建、禁用、启用和密钥轮换。
- 服务端支持登录限流和 Agent 上报基础限流。
- 服务端支持首次配置状态、密码初始化、端口保存、证书上传、证书到期解析、数据库压缩下载和登录后重新打开配置引导。
- 服务端支持 React/Vite 构建目录托管，并保留内置 HTML 面板作为回退；fallback 面板也已覆盖总览、设备、GPU、设置、深浅主题、GPU 趋势图和服务设置入口。
- Web 面板支持首次配置引导、密码登录、退出、卡片式多机多卡 GPU Fleet 总览、GPU 详情趋势图、趋势图悬浮读数、同设备 GPU 边框同色、设备管理、GPU 当前状态、GPU 进程快照、24 小时统计、深浅主题切换和服务设置页。
- Agent 支持 GPU 进程快照采集和上报。
- Agent 支持本地离线样本队列，网络失败时缓存 GPU 样本并限制本地队列大小。
- 已提供 Windows Service 和 Linux systemd 安装/卸载脚本。
- 已提供 GPU 利用率统计 API 和最新进程快照 API。
- 已添加静态面板路由测试，覆盖 SPA fallback、API 404、目录越界防护，以及内置 fallback 面板的设置页、趋势图、离线蒙版和旧进度条回归检查。
- 已提供 Chrome/Edge CDP 前端验证脚本，覆盖登录、刷新会话恢复、GPU Fleet 卡片面板、GPU 详情趋势图、深浅主题切换和持久化、设备页、设置页、移动端总览、移动端 GPU 页、截图和移动端横向溢出检查；演示验证可强制检查 5 张 GPU 卡片、离线蒙版和同设备双卡。
- 已提供 `scripts/seed-demo-data.mjs`，用于初始化 4 台设备、5 块 GPU 的本地演示数据，其中包含一个双卡设备和一个离线设备。

## 运行边界

当前 MVP 不依赖外部数据库，适合先在单机公网服务端上验证链路。若设备数、保留时间或查询复杂度提高，应引入 VictoriaMetrics 作为时序后端。

服务端仍然不提供任何客户端控制能力。管理接口只影响服务端的设备记录和认证状态，不会修改客户端本地配置。
