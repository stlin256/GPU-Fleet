# 产品细节

## 产品定位

GPUFleet 是一个轻量 GPU 资源观察平台，用于把分散在不同网络环境中的 NVIDIA GPU 设备聚合到一个公网 Web 面板中。

它关注三类问题：

- 当前有哪些 GPU 在线，它们是否健康。
- 每张 GPU 的利用率、显存、温度、功耗、进程占用如何变化。
- 一段时间内 GPU 使用率、空闲率、峰值、异常次数如何统计。

## 目标用户

- 小型实验室、个人工作站集群、训练机器池。
- 不希望搭建完整 Prometheus/Grafana/Exporter 体系的团队。
- 设备分布在家庭宽带、办公室、云主机、远程机房等不同网络下的用户。

## 核心约束

- 服务端位于公网，可被客户端访问。
- 客户端可能位于 NAT、动态 IP、内网或临时网络。
- 服务端不能主动访问客户端。
- 服务端不能对客户端产生配置影响或执行影响。
- 依赖尽可能少，能用单机部署完成第一版。
- 数据库必须控制磁盘占用，默认至少预留 800MiB 空闲空间。

## MVP 功能范围

### 设备总览

- 在线设备数。
- 离线设备数。
- GPU 总数。
- 总显存、已用显存。
- 当前平均 GPU 利用率。
- 高温 GPU 数。
- 最近 24 小时异常事件数。

### 设备列表

- 设备别名。
- 操作系统。
- Agent 版本。
- GPU 数量。
- 最近上报时间。
- 在线/离线状态。
- 最近错误摘要。

### GPU 详情

- GPU 型号。
- GPU UUID 或脱敏 ID。
- 驱动版本。
- CUDA 版本，后续通过 NVML 或驱动侧信息补齐；当前 `nvidia-smi --query-gpu` 本机不可直接查询该字段。
- VBIOS、Compute 能力、显示状态、驱动模型。
- 显存总量。
- GPU 利用率曲线。
- 显存使用曲线。
- 温度曲线。
- 功耗曲线。
- 风扇转速、图形/显存/SM/视频时钟、P-State、PCIe 当前和最大链路信息。
- ECC/MIG 状态，若设备支持。

### 进程视图

- 当前占用 GPU 的进程。
- PID。
- 进程名。
- 占用显存。
- 可选用户名。
- 可选命令行摘要。

默认不采集用户名和命令行，避免泄露敏感信息。

### 统计记录

- 按设备、GPU、时间范围统计平均利用率。
- 统计峰值温度、峰值功耗、峰值显存。
- 统计空闲时长和空闲占比。
- 统计离线次数和离线时长。
- 支持 1 小时、24 小时、7 天、30 天和自定义范围。

### 告警

MVP 只做服务端本地告警记录，不做复杂通知系统。

默认规则：

- 设备超过 3 个采集周期未上报，标记离线。
- 温度超过阈值，记录高温事件。
- 显存使用率超过阈值，记录显存压力事件。
- 采集失败连续超过阈值，记录 Agent 异常事件。

## 非目标

第一版不做：

- 任务调度。
- 训练任务管理。
- 远程 Shell。
- 远程杀进程。
- GPU 设置修改。
- 多租户计费。
- Kubernetes 集群管理。
- 复杂可视化编辑器。

这些功能和“服务端不能影响客户端”的安全边界冲突，后续也应谨慎评估。
