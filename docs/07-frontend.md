# Web 前端设计

## 设计目标

- 多端适配：桌面、平板、手机都可用。
- 信息密度适中：桌面端适合监控大盘，移动端适合快速检查状态。
- 现代灵动：轻量动效、清晰层级、实时状态反馈。
- 不依赖 Grafana，产品体验由 GPUFleet 自身控制。

## 技术栈

当前已实现：

- React。
- Vite。
- TypeScript。
- lucide-react。
- TanStack Query。
- Apache ECharts。

暂未引入 Tailwind CSS、shadcn/ui 和 date-fns，避免依赖膨胀。当前样式使用项目内 CSS，图标、查询和图表依赖由包管理。

构建产物提交在 `web/dist`，服务端默认通过 `-web-dir web/dist` 托管。若构建目录不存在，服务端会回退到内置轻量 HTML 面板。

## 信息架构

### 当前已实现页面

当前 React 面板提供四个主视图：

- 总览：多机多卡 GPU Fleet 聚合面板、设备状态、利用率分布、GPU 进程快照和 24 小时统计。
- 设备：创建设备、显示一次性密钥、禁用/启用设备、轮换密钥。
- GPU：展示单卡详细运行字段、当前利用率图表、设备状态、GPU 进程快照和统计面板；后续会扩展为单卡详情路由。
- 设置：展示服务端安全边界和磁盘保护状态。

### 总览页

桌面端：

- 顶部 Fleet Command：在线设备、GPU 总数、忙碌 GPU、高温 GPU、显存占用和磁盘保护。
- 主区域：GPU Fleet 聚合表，以设备/GPU 为行，展示利用率、显存、温度、功耗、PCIe 链路和健康状态。
- 右侧：当前利用率分布和设备状态。
- 下方：GPU 进程快照和 24 小时统计。

移动端：

- 顶部 Fleet Command 自动收敛为两列或单列。
- GPU Fleet 聚合表改为卡片式行布局，避免横向溢出。
- 设备和进程面板纵向排列。

### GPU 页

- GPU 卡片展示显存空闲/保留、显存利用率、温度上限、功耗上限、风扇、图形/显存/SM/视频时钟、P-State、PCIe 当前/最大链路、Compute、显示状态、驱动模型、VBIOS、ECC/MIG 和时钟限速原因。
- 总览页保留简洁巡检视图，GPU 页承载更完整的单卡细节，避免第一屏信息堆砌。

### 设备页

- 注册新设备并返回一次性设备密钥。
- 展示设备 ID、别名、主机名、Agent 版本和启用状态。
- 禁用或启用设备。禁用只影响服务端认证，不修改客户端配置。
- 轮换设备密钥。旧密钥立即失效，新密钥只显示一次。

后续增强：

- 按在线状态、OS、标签筛选。
- 搜索设备别名。
- 显示最后上报时间和最近错误摘要。

### 后续 GPU 详情页

- 当前状态摘要。
- 分时图表。
- 进程占用表。
- 历史统计。
- 告警记录。

### 后续统计页

- 时间范围选择。
- 设备/GPU 多选。
- 平均利用率排行。
- 空闲率排行。
- 峰值温度排行。
- CSV 导出。

### 设置页

- 服务端不会控制客户端的安全边界说明。
- Agent 接入认证能力说明。
- Web 会话能力说明。
- 当前空闲磁盘、保留阈值和回收策略。

后续增强：

- 管理员账号。
- 告警阈值。
- 数据保留策略编辑。

| `>1024px` | 固定侧边栏、多列仪表盘 |

- 总览。

- 设置。

## 视觉风格

- 背景使用中性浅色或深色主题，不做大面积单一蓝紫渐变。
- 顶栏提供深浅主题切换，使用 `localStorage` 持久化；刷新后保留主题。
- 状态颜色明确：绿色在线、黄色警告、红色异常、灰色离线。
- 卡片圆角不超过 8px。
- 图表突出数据，不使用过多装饰。
- 使用 lucide 图标表达动作，例如刷新、筛选、下载、设置。

## 交互细节

- 图表 hover 显示精确数值。
- 时间范围切换不整页刷新。
- 设备离线状态即时显著。
- 表格支持排序和筛选。
- 请求失败展示可恢复错误。
- 磁盘保护触发时顶部显示固定告警条。

## 实时更新

MVP 使用轮询：

- 总览页：10 秒。
- 统计数据：30 秒。

后续可升级为 Server-Sent Events。暂不使用 WebSocket，减少复杂度。

## 浏览器验证

已提供 `scripts/verify-frontend-chrome.mjs`，使用本机 Chrome/Edge 的 CDP 能力完成无额外 npm 依赖的端到端浏览器验证。脚本会登录 Web 面板、验证刷新后 Cookie 会话恢复、检查 GPU Fleet 聚合表、检查深浅主题切换和持久化、检查设备页、检查移动端总览和 GPU 页、确认没有横向溢出，并输出桌面/移动端截图和 `result.json`。
