# Web 前端设计

## 设计目标

- 多端适配：桌面、平板、手机都可用。
- 信息密度适中：桌面端适合监控大盘，移动端适合快速检查状态。
- 现代灵动：轻量动效、清晰层级、实时状态反馈。
- 不依赖 Grafana，产品体验由 GPUFleet 自身控制。

## 技术栈

当前已实现：

- React。
- Vite。
- TypeScript。
- lucide-react。
- TanStack Query。
- Apache ECharts。

暂未引入 Tailwind CSS、shadcn/ui 和 date-fns，避免依赖膨胀。当前样式使用项目内 CSS，图标、查询和图表依赖由包管理。

构建产物提交在 `web/dist`，服务端默认通过 `-web-dir web/dist` 托管。若构建目录不存在，服务端会回退到内置轻量 HTML 面板。

## 信息架构

### 当前总览页

桌面端：

- 顶部状态条：在线设备、GPU 总数、平均利用率、异常数。
- 主区域：GPU 当前状态卡片。
- 右侧：设备列表和 GPU 进程快照。
- 下方：24 小时统计表。
- 图表：ECharts GPU 利用率柱状图。

移动端：

- 顶部状态卡自动单列。
- GPU 状态列表。
- 设备和进程面板纵向排列。

### 设备页

- 设备列表。
- 按在线状态、OS、标签筛选。
- 搜索设备别名。
- 显示最后上报时间。

### 后续 GPU 详情页

- 当前状态摘要。
- 分时图表。
- 进程占用表。
- 历史统计。
- 告警记录。

### 后续统计页

- 时间范围选择。
- 设备/GPU 多选。
- 平均利用率排行。
- 空闲率排行。
- 峰值温度排行。
- CSV 导出。

### 后续设置页

- 管理员账号。
- 设备注册和密钥轮换。
- 告警阈值。
- 数据保留策略展示。
- 服务端磁盘状态。

## 响应式布局

断点建议：

| 断点 | 布局 |
| --- | --- |
| `<640px` | 单列、底部导航、图表横向滚动或简化 |
| `640-1024px` | 双列卡片、侧边栏折叠 |
| `>1024px` | 固定侧边栏、多列仪表盘 |

移动端底部导航：

- 总览。
- 设备。
- 告警。
- 设置。

桌面端左侧导航：

- 总览。
- 设备。
- GPU。
- 统计。
- 告警。
- 设置。

## 视觉风格

- 背景使用中性浅色或深色主题，不做大面积单一蓝紫渐变。
- 状态颜色明确：绿色在线、黄色警告、红色异常、灰色离线。
- 卡片圆角不超过 8px。
- 图表突出数据，不使用过多装饰。
- 使用 lucide 图标表达动作，例如刷新、筛选、下载、设置。

## 交互细节

- 图表 hover 显示精确数值。
- 时间范围切换不整页刷新。
- 设备离线状态即时显著。
- 表格支持排序和筛选。
- 请求失败展示可恢复错误。
- 磁盘保护触发时顶部显示固定告警条。

## 实时更新

MVP 使用轮询：

- 总览页：10 秒。
- 统计数据：30 秒。

后续可升级为 Server-Sent Events。暂不使用 WebSocket，减少复杂度。
