# Web 前端设计

## 设计目标

- 多端适配：桌面、平板、手机都可用。
- 信息密度适中：桌面端适合监控大盘，移动端适合快速检查状态。
- 现代灵动：轻量动效、清晰层级、实时状态反馈。
- 不依赖 Grafana，产品体验由 GPUFleet 自身控制。

## 技术栈

推荐：

- React。
- Vite。
- TypeScript。
- Tailwind CSS。
- shadcn/ui。
- lucide-react。
- TanStack Query。
- Apache ECharts。
- date-fns。

保持克制，不引入过多 UI 框架。组件库负责基础控件，业务图表和布局由项目控制。

## 信息架构

### 总览页

桌面端：

- 顶部状态条：在线设备、GPU 总数、平均利用率、异常数。
- 主区域：GPU 利用率热力矩阵。
- 右侧：最近告警、离线设备。
- 下方：全局利用率、显存、温度趋势。

移动端：

- 顶部折叠状态卡。
- GPU 状态列表。
- 最近告警。
- 点击进入详情页。

### 设备页

- 设备列表。
- 按在线状态、OS、标签筛选。
- 搜索设备别名。
- 显示最后上报时间。

### GPU 详情页

- 当前状态摘要。
- 分时图表。
- 进程占用表。
- 历史统计。
- 告警记录。

### 统计页

- 时间范围选择。
- 设备/GPU 多选。
- 平均利用率排行。
- 空闲率排行。
- 峰值温度排行。
- CSV 导出。

### 设置页

- 管理员账号。
- 设备注册和密钥轮换。
- 告警阈值。
- 数据保留策略展示。
- 服务端磁盘状态。

## 响应式布局

断点建议：

| 断点 | 布局 |
| --- | --- |
| `<640px` | 单列、底部导航、图表横向滚动或简化 |
| `640-1024px` | 双列卡片、侧边栏折叠 |
| `>1024px` | 固定侧边栏、多列仪表盘 |

移动端底部导航：

- 总览。
- 设备。
- 告警。
- 设置。

桌面端左侧导航：

- 总览。
- 设备。
- GPU。
- 统计。
- 告警。
- 设置。

## 视觉风格

- 背景使用中性浅色或深色主题，不做大面积单一蓝紫渐变。
- 状态颜色明确：绿色在线、黄色警告、红色异常、灰色离线。
- 卡片圆角不超过 8px。
- 图表突出数据，不使用过多装饰。
- 使用 lucide 图标表达动作，例如刷新、筛选、下载、设置。

## 交互细节

- 图表 hover 显示精确数值。
- 时间范围切换不整页刷新。
- 设备离线状态即时显著。
- 表格支持排序和筛选。
- 请求失败展示可恢复错误。
- 磁盘保护触发时顶部显示固定告警条。

## 实时更新

MVP 使用轮询：

- 总览页：10 秒。
- GPU 详情页：10 秒。
- 告警页：30 秒。

后续可升级为 Server-Sent Events。暂不使用 WebSocket，减少复杂度。
