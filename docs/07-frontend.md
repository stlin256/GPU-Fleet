# Web 前端设计

## 设计目标

- 多端适配：桌面、平板、手机都可用。
- 信息密度适中：桌面端适合监控大盘，移动端适合快速检查状态。
- 现代灵动：轻量动效、清晰层级、实时状态反馈。
- 不依赖 Grafana，产品体验由 GPUFleet 自身控制。

## 技术栈

当前已实现：

- React。
- Vite。
- TypeScript。
- lucide-react。
- TanStack Query。
- Apache ECharts。

暂未引入 Tailwind CSS、shadcn/ui 和 date-fns，避免依赖膨胀。当前样式使用项目内 CSS，图标、查询和图表依赖由包管理。

构建产物提交在 `web/dist`，服务端默认通过 `-web-dir web/dist` 托管。若构建目录不存在，服务端会回退到内置 HTML 面板；该 fallback 面板也提供总览、设备、GPU、设置、深浅主题、GPU 趋势小图和服务设置入口，避免启动目录错误时退回空设置页或旧进度条体验。

## 信息架构

### 当前已实现页面

当前 React 面板提供四个主视图：

- 总览：卡片式多机多卡 GPU Fleet 聚合面板、设备状态、利用率分布、GPU 进程快照和 24 小时统计。
- 设备：创建设备、显示一次性密钥、修改设备名称、禁用/启用设备、删除设备、轮换密钥，危险操作使用应用内确认弹窗。
- GPU：展示单卡详细运行字段、2x2 历史趋势图、设备状态、GPU 进程快照和统计面板；24 小时统计可展开多个 GPU 查看 24 小时曲线。
- 设置：提供服务状态、密码更改、访问端口配置、磁盘预留空间、HTTPS 证书上传、证书到期显示、数据库下载、在线更新、手动重启服务、访客功能、配置引导入口、版本号和 Changelog。
- 访客总览：`/guest` 只展示脱敏总览和 GPU 卡片曲线，不展示 GPU 进程、24 小时统计或管理入口。
- i18n：首次配置先选择语言，设置页可修改语言；当前支持简体中文和 English，文案由 `web/src/i18n.ts` 的可扩展词表维护。

### 总览页

桌面端：

- 顶部 Fleet Command：在线设备、GPU 总数、忙碌 GPU、高温 GPU、总显存用量和总功耗。
- 主区域：GPU Fleet 卡片网格，每块 GPU 独立成卡，卡内用 2x2 历史趋势小图展示利用率、显存、温度和功耗，并保留 PCIe 链路和健康状态。
- 顶部汇总指标中，总显存用量、总功耗等卡片可展示迷你曲线，避免只显示瞬时数值。
- 右侧：当前利用率分布和设备状态。
- 下方：GPU 进程快照和 24 小时统计；24 小时统计行可展开多个 GPU，查看利用率、显存、温度和功耗曲线。

移动端：

- 顶部 Fleet Command 自动收敛为两列或单列。
- GPU Fleet 卡片网格收敛为单列，卡内历史趋势图在手机宽度仍保持 2x2，并压缩图表尺寸减少滚动。
- 设备和进程面板纵向排列。

### GPU 页

- GPU 详情卡使用 2x2 历史趋势图展示利用率、显存、温度和功耗，避免只看当前瞬时值。
- 趋势小图支持鼠标悬浮读数，显示当前指标名称、采样时间和数值。
- 同一设备下的 GPU 卡片使用同一色系外框线，方便在多机多卡网格里快速识别设备归属。
- GPU 卡片展示显存空闲/保留、显存利用率、温度上限、功耗上限、风扇、图形/显存/SM/视频时钟、P-State、PCIe 当前/最大链路、Compute、显示状态、驱动模型、VBIOS、ECC/MIG 和时钟限速原因。
- 总览页保留简洁巡检视图，GPU 页承载更完整的单卡细节，避免第一屏信息堆砌。

### 设备页

- 注册新设备并返回一次性设备密钥。
- 展示设备 ID、别名、主机名、Agent 版本和启用状态。
- 修改设备显示名称。
- 禁用或启用设备。禁用只影响服务端认证，不修改客户端配置。
- 删除设备。删除只移除服务端记录、最新 GPU 缓存和最新进程快照，不连接客户端。
- 轮换设备密钥。旧密钥立即失效，新密钥只显示一次。
- 禁用、启用、删除和密钥轮换使用应用内确认弹窗。

### 设置页

- 服务状态：当前协议、当前监听地址、配置端口、证书到期日期、磁盘预留和是否需要重启。
- 密码更改：只使用密码作为 Web 凭据，不需要账户名；登录后记住当前浏览器设备 30 天。
- 端口配置：保存下一次启动要使用的访问端口；当前进程不热切换监听端口。
- 语言设置：保存服务端界面语言，立即影响当前面板和后续浏览器会话。
- HTTPS 证书：上传证书 PEM 和私钥 PEM；无证书时使用 HTTP，有证书并重启后使用 HTTPS。
- 数据库下载：下载运行数据库压缩包，不包含证书私钥。
- 磁盘预留空间：修改服务端拒绝写入前需要保留的最小空闲空间，保存后立即生效。
- 在线更新：检查服务端 Git upstream，显示当前/远端提交、behind、二进制过期状态和缓存检查时间；支持代理地址，点击“更新”后先显示全屏确认弹窗，再执行构建、拉取和自动重启。
- 手动重启服务：确认后调度服务端重启，页面全屏等待服务恢复，恢复后弹出重启成功提示。
- 访客功能：开启登录页访客入口，查看访客记录弹窗；记录包含浏览器指纹摘要、语言、平台、屏幕、时区、远端地址和 User-Agent。
- 版本与变更：显示项目品牌 Logo、作者 `stlin256`、仓库地址 `https://github.com/stlin256/GPU-Fleet`、当前版本号、提交、构建时间和当前版本 Changelog；点击“更多更新记录”打开完整 Changelog 弹窗。
- 配置引导：登录后可重新打开首次配置流程，集中调整密码、端口、语言和证书；重新打开时复用与首次启动一致的全屏品牌化引导，并预填已有配置。

### 访客总览

- 登录页在服务端开启访客功能后显示居中的“访客访问”按钮。
- 访客页使用 `/guest` URL，顶部标题为“访客总览”。
- 访客模式只展示脱敏总览、设备名称/在线状态和 GPU 卡片曲线。
- 访客模式不展示 GPU 进程、24 小时统计、设备真实 ID、主机名、Agent 版本、驱动版本、GPU UUID、VBIOS 或管理入口。
- 访客模式移动端不显示底部导航栏，减少只读视图的占用。
- 访客曲线使用独立的 `/api/v1/guest/gpus/{gpu_id}/series` 接口；访客功能关闭后该接口返回 `403`。

### i18n

- 语言配置存储在服务端 `metadata.json` 的 `service.language` 字段中。
- React 面板使用 `web/src/i18n.ts` 维护词表，新增用户可见文案应优先加入词表并通过 `t()` 调用。
- 动态文案可在 `enPatterns` 中维护模板匹配，避免设备名、GPU 名、提交号和 URL 被误翻译。
- DOM 翻译兜底用于覆盖旧组件和动态渲染文本，但不替代显式词表维护。
- 内置 fallback 面板也有轻量词表，避免缺少 `web/dist` 时退回单语言体验。

## 视觉风格

- 背景使用中性浅色或深色主题，不做大面积单一蓝紫渐变。
- 顶栏提供深浅主题切换，使用 `localStorage` 持久化；刷新后保留主题。
- 状态颜色明确：绿色在线、黄色警告、红色异常、灰色离线。
- 卡片圆角不超过 8px。
- 图表突出数据，不使用过多装饰。
- 使用 lucide 图标表达动作，例如刷新、筛选、下载、设置。
- 品牌 Logo 使用 `web/public/brand/gpufleet-logo.svg`，侧边栏、登录页、配置引导和设置页复用同一 SVG 标识。
- 页面切换、卡片进入、按钮按压、表单焦点、图表绘制、提示气泡和离线蒙版都有轻量动效；遵守 `prefers-reduced-motion`，系统要求减少动态时会自动收敛。

## 交互细节

- 图表 hover 显示精确数值和采样时间。
- 视图切换使用短进入动效，避免切页突兀。
- 卡片、设备行、统计行和设置操作项提供 hover/press 反馈。
- 认证后的移动端使用底部固定导航，预留安全区和底部内容间距，避免遮挡最后一屏内容；访客模式移动端隐藏底部导航。
- 所有确认、更新进度、更新完成、重启、访客记录、Changelog 和 fallback 面板弹窗都挂载到全屏视口层，并使用一致的背景模糊，避免只覆盖当前卡片或当前 tab 区域。
- 时间范围切换不整页刷新。
- 设备离线状态即时显著。
- 表格支持排序和筛选。
- 请求失败展示可恢复错误。
- 磁盘保护触发时顶部显示固定告警条。

## 浏览器验证

已提供 `scripts/verify-frontend-chrome.mjs`，使用本机 Chrome/Edge 的 CDP 能力完成无额外 npm 依赖的端到端浏览器验证。脚本会登录 Web 面板、验证刷新后 Cookie 会话恢复、检查 GPU Fleet 卡片面板和 2x2 历史趋势图、检查趋势图悬浮读数、检查同设备 GPU 边框同色、检查深浅主题切换和持久化、检查设备页、检查设置页操作入口、在线更新入口、品牌 Logo、仓库署名、版本号和 Changelog、检查移动端总览和 GPU 页、确认底部导航固定和无横向溢出，并输出桌面/移动端截图和 `result.json`。

演示场景可配合 `scripts/seed-demo-data.mjs` 生成 4 台设备、5 块 GPU 的数据，其中 `rig-dual` 包含 2 块 GPU，`rig-offline` 用于验证离线灰色蒙版。验证脚本支持 `--min-fleet-cards`、`--require-offline-mask` 和 `--require-dual-device`，用于对这类演示数据做强断言。
