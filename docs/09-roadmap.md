# 实施路线图

## Phase 0：文档和仓库

- 明确产品边界。
- 明确安全模型。
- 明确存储模型和磁盘保护策略。
- 初始化 Git 仓库。

## Phase 1：最小可运行链路

- 编写 Windows/Linux Agent。
- 使用 `nvidia-smi` 采集基础 GPU 指标。
- 实现 HMAC 上报。
- 实现服务端接收 API。
- 写入 VictoriaMetrics。
- SQLite 保存设备元数据。
- Web 展示设备列表和当前指标。

## Phase 2：历史曲线和统计

- GPU 详情页。
- 最近 1 小时、24 小时、7 天曲线。
- 平均利用率、峰值温度、峰值功耗统计。
- 设备离线检测。
- Agent 本地队列。

## Phase 3：安全和运维完善

- 管理员登录。
- 设备密钥轮换。
- 请求限流。
- 审计日志。
- 磁盘保护告警。
- Windows Service 和 systemd 安装脚本。

## Phase 4：采集增强

- NVML 采集替换 `nvidia-smi`。
- 进程快照。
- MIG/ECC 信息。
- PCIe、时钟、风扇等更多字段。
- Linux 多发行版验证。

## Phase 5：体验增强

- 深色主题。
- 响应式图表优化。
- CSV 导出。
- 告警规则配置。
- Server-Sent Events 实时刷新。

## 优先级原则

先保证：

- 数据链路正确。
- 安全边界清晰。
- 磁盘不会写满。
- 单机部署简单。

再增加：

- 更多指标。
- 更丰富图表。
- 更复杂统计。
