# 状态与路线图

本文按当前仓库状态记录已经落地的能力和后续增强方向，避免把已完成内容继续写成 MVP 计划项。

## 已落地

### 数据链路

- Windows/Linux Agent 使用 `nvidia-smi` 只读采集 NVIDIA GPU 指标和可选进程快照。
- Agent 主动通过 HTTP/HTTPS 上报，不监听端口。
- Agent 支持本地离线样本队列、回放和队列容量限制。
- 服务端使用 HMAC、时间戳和原子 nonce 校验上报，并限制请求体大小和上报频率。
- 服务端写入 gzip JSONL 小时分段，维护最新 GPU 状态、进程快照、内存索引和 rollup。

### Web 与设备管理

- React 面板已覆盖总览、GPU、设备、设置和访客总览。
- 总览和 GPU 页支持多机多卡 Fleet 卡片、2x2 趋势图、悬浮读数、离线蒙版、同设备边框同色和移动端适配。
- 统计面板支持 1H、6H、24H、7D 和 30D 范围，长范围查询使用 rollup 边界容错。
- 能源页支持 24H、7D 和 30D kWh、电费估算、功率/热状态趋势、GPU 能耗排行、高温、限速和空转高耗诊断；该能力只基于现有只读指标派生。
- 设备页支持注册、一次性密钥、改名、禁用/启用、删除和轮换密钥。
- 设置页支持语言、密码、端口、HTTPS 证书、磁盘预留、能耗展示参数、数据库下载、诊断包下载、在线更新、手动重启、访客功能、版本信息和 Changelog。
- 内置 fallback 面板覆盖核心视图和设置入口，避免 `web/dist` 缺失时退回空白或旧体验。

### 安全与运维

- Web 登录支持 PBKDF2 密码哈希、30 天 Cookie 会话、登录限流和递进锁定。
- 管理写接口有 Origin/Referer 同源校验。
- 访客模式只暴露脱敏总览和访客 GPU 曲线，不暴露管理接口、进程、真实设备 ID、主机名、驱动版本、GPU UUID 或 VBIOS。
- 服务端在线更新默认每 30 分钟检查 upstream；手动和自动更新都使用固定 Git/Go 流程、供应链来源校验、远端构建、fast-forward 拉取和自动重启。
- 更新重启流程会保留 `.bak`；Linux systemd 场景先同步替换二进制再由 systemd 拉起新进程，非 systemd 场景先替换二进制再等待旧进程退出，并在启动时补救遗留 `.next` 替换。
- Linux 服务端提供备份与恢复脚本，恢复需要显式确认，并保留回滚目录。
- 设置页可下载只读诊断包，用于排查版本、运行时、磁盘、分段、设备、GPU、进程、更新缓存和审计摘要。

### 验证

- Go 单元测试覆盖服务端路由、fallback 面板、登录限流、设备生命周期、在线更新、诊断包、访客接口、统计查询和更新重启脚本关键逻辑。
- `scripts/verify-frontend-chrome.mjs` 使用 Chrome/Edge CDP 做浏览器级 smoke 验证，覆盖登录、会话恢复、图表、主题、移动端、设置入口、诊断包入口、关键弹窗和截图非空。
- `scripts/seed-demo-data.mjs` 可生成 4 台设备、5 块 GPU 的演示数据，其中包含双卡设备和离线设备。

## 后续增强

### 数据后端

- VictoriaMetrics 适配：当设备数、保留时间或查询复杂度继续提高时，用单节点 VictoriaMetrics 承载时序数据。
- SQLite 元数据适配：替代 JSON 元数据文件，提高并发写入、迁移和查询能力。

### 采集增强

- NVML 采集：降低进程调用开销，提供更完整的错误码和更一致的 MIG/ECC/进程字段。
- Linux 多发行版和更多 GPU 型号长期验证。
- CUDA 版本等当前 `nvidia-smi --query-gpu` 路径不直接提供的只读字段补齐。

### 产品能力

- 可配置告警规则和外部通知。
- CSV/JSON 报表导出。
- 能耗报表导出、跨 30D 真实长期运行样本校准，以及更细的设备/机房能耗分组。
- 更丰富的设备筛选、标签和搜索。
- Server-Sent Events 实时刷新，减少轮询延迟。

## 优先级原则

后续仍优先保持三条边界：

- 数据链路正确，长时间运行可诊断。
- 服务端不能成为客户端远程控制通道。
- 默认部署不因外部数据库、复杂依赖或磁盘写满而失去可用性。
