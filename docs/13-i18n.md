# i18n

## Scope

GPUFleet currently supports:

- `zh-CN`: Simplified Chinese.
- `en-US`: English.

The selected language is stored in `metadata.json` under `service.language`. Existing deployments without this field default to `zh-CN`.

## Runtime Flow

- `GET /api/v1/setup/status` returns `service.language`.
- First setup submits `language` together with password, port, and optional certificate data.
- `GET /api/v1/overview` returns `service.language`, so authenticated browsers converge on the server-side language.
- `GET /api/v1/guest/status` and guest overview responses expose the minimal language/scheme state needed for the login and guest views without exposing admin-only data.
- `POST /api/v1/admin/language` updates only the language and does not require a service restart.
- Reopened setup also accepts `language`.

## Frontend Framework

The React dashboard keeps translations in `web/src/i18n.ts`.

Maintenance rules:

- Add user-facing text to the `en` dictionary using the Chinese source text as the stable key.
- Add dynamic templates to `enPatterns` when text contains runtime values.
- Prefer `t("中文文案")` in new components.
- The DOM translation fallback remains as a safety net for legacy or dynamic text, but new code should still call `t()`.
- Do not translate identifiers, device IDs, GPU names, file paths, URLs, commits, or metric units.
- Energy-page labels, diagnostics, range controls, and display-setting form text should use the same dictionary path as the rest of the React dashboard.
- Dialog copy for update, restart, guest records, changelog, and setup should use the same dictionary path as page content. Browser-native file input text is wrapped by project UI to avoid locale leaks.

## Built-In Fallback Dashboard

When `web/dist` is missing, the server serves `internal/server/web.go`. It includes a lightweight dictionary and DOM translation pass so the fallback dashboard also respects `service.language` for the main user-facing UI.
